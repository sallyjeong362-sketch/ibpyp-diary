# 나의 영어 성찰 일기 (English Reflection Diary)

유치원부터 고등학생까지 함께 쓸 수 있는 자기 성찰 영어일기 앱입니다. 그림일기처럼 사진 한 장을 위에 올리고, 아래에 3줄(이상)로 오늘을 영어로 기록합니다.

- `index.html` — 앱 본체 (학생/교사 역할 분리, 학급 코드, 주제 추천, 표현 템플릿)
- `firebase-config.js` — Firebase 프로젝트 설정 (아래 안내에 따라 채워야 함)
- `firestore.rules` — Firebase 콘솔에 붙여넣을 Firestore 보안 규칙

## 사용법

- **학생**: 링크를 열고 "학생으로 시작하기" → 학급 코드 + 이름만 입력. 회원가입/로그인 없음. "내 일기" / "반 친구들" / "우리 반 정원" 탭으로 같은 반 친구들의 일기도 볼 수 있고, 서로 댓글을 남길 수 있어요. 단, 일기 원문은 작성한 본인만 수정·삭제할 수 있습니다.
- **성장 트래커**: 일기를 쓸 때마다 씨앗 🌰 → 새싹 🌱 → 떡잎 🌿 → 어린나무 🪴 → 나무 🌳 → 무성한 나무 🌲 → 열매 맺은 나무 🌳🍎 → 만개한 나무 🌳🌸 순서로 자라나요. 내 일기 목록 위에 항상 보이고, "우리 반 정원" 탭에서는 반 친구들 나무를 한눈에 볼 수 있어요.
- **선생님**: "선생님·관리자" → "학급 만들기"에서 학급 이름과 비밀번호를 정하면 학급 코드가 생성됩니다. 이후 "학급 열기"에서 코드+비밀번호로 들어가면 학생들이 제출한 모든 일기를 달력·학생별로 모아보고, 확인 표시와 코멘트를 남길 수 있어요. "🌳 우리 반 성장 정원 보기"로 반 전체의 성장 현황도 한 화면에서 볼 수 있어요 (교실 화면에 띄워서 공유하기 좋아요).
- **기간별 인쇄 · PDF**: 대시보드의 "🖨️ 기간별 인쇄 · PDF 만들기"에서 학생(또는 전체 학생)과 기간을 고르면 해당 일기들을 한 문서로 모아줍니다. "인쇄 / PDF로 저장" 버튼을 누르면 브라우저 인쇄 대화상자가 뜨는데, 프린터 대신 "PDF로 저장"을 고르면 파일로 저장돼서 이메일이나 메신저로 학부모/학생에게 전송할 수 있어요.
- **AI 첨삭 (문법 · 철자)**: 선생님이 학생 일기를 검토하는 화면(대시보드 → 일기 클릭)에서 "✨ AI 첨삭 받기"를 누르면 [LanguageTool](https://languagetool.org)의 무료 문법·철자 검사 API로 오류를 찾아 밑줄과 함께 수정 제안을 보여줍니다. 마음에 드는 제안은 "코멘트에 추가"를 눌러 선생님 코멘트에 바로 반영할 수 있어요 (일기 원문은 여전히 학생 본인만 고칠 수 있고, 첨삭 결과는 어디까지나 선생님이 코멘트를 쓸 때 참고하는 용도입니다). 별도 API 키나 설정 없이 바로 동작하지만, 인터넷 연결과 LanguageTool 서비스 상태에 따라 가끔 실패할 수 있어요(그럴 땐 안내 메시지가 뜨고 잠시 후 다시 시도하면 됩니다).
- `firebase-config.js`를 설정하지 않으면 **데모 모드**로 동작합니다(브라우저 탭에서만 임시 저장, 새로고침해도 유지되지만 탭을 닫으면 사라짐). 실제 반 운영 전에 기능을 미리 눌러볼 때 유용합니다.

## 단계별 기능

- 🐣 유치원·초1-2: 매일 3문장 고정 (What happened → How I felt → What I learned)
- 📘 초3-6학년: 3~6줄
- 📗 중학생: 3~10줄, 긴 글로 전환 가능
- 📕 고등학생: 3줄부터 에세이 형식의 긴 글까지 자유

## 실제 학급에서 쓰기 위한 설정 (최초 1회)

### 1. Firebase 프로젝트 만들기

1. https://console.firebase.google.com 에서 무료로 새 프로젝트를 만듭니다.
2. 왼쪽 메뉴 **Firestore Database** → "데이터베이스 만들기" → 테스트 모드로 시작(위치는 가까운 리전 선택).
3. Firestore **규칙(Rules)** 탭을 열고, 이 저장소의 `firestore.rules` 내용을 그대로 붙여넣은 뒤 "게시".
4. 왼쪽 메뉴 **Authentication** → "시작하기" → **Sign-in method** 탭 → **익명(Anonymous)** 선택 → 사용 설정. (학생/교사에게 로그인 화면을 보여주지 않으면서도, 각자의 일기를 "본인만 수정·삭제 가능"하게 구분하는 데 필요합니다.)
5. 프로젝트 개요 옆 톱니바퀴 → **프로젝트 설정** → 아래로 스크롤해서 "내 앱" → 웹 앱 추가(`</>`) → 이름은 아무거나(예: diary) → "앱 등록".
6. 화면에 나오는 `firebaseConfig` 객체 값을 복사합니다.

### 2. firebase-config.js 채우기

이 저장소의 `firebase-config.js` 파일을 열어 `YOUR_...` 로 되어 있는 값들을 1번에서 복사한 값으로 교체하고 커밋합니다. (이 값들은 공개되어도 안전한 클라이언트 식별자입니다. 실제 접근 제어는 Firestore 규칙 + 학급 코드/비밀번호가 담당합니다.)

### 3. GitHub Pages로 배포하기

1. GitHub 저장소 → **Settings** → **Pages**.
2. "Build and deployment" → Source: **Deploy from a branch** 선택.
3. Branch: `main`, 폴더: `/ (root)` 선택 후 저장.
4. 몇 분 뒤 `https://<사용자명>.github.io/ibpyp-diary/` 링크가 활성화됩니다. 이 링크를 학생/선생님 모두에게 공유하면 됩니다.

## 보안에 대한 참고

이 앱은 이름/비밀번호 로그인 없이 "학급 코드 + 교사 비밀번호"만으로 동작하는 가벼운 학급용 도구입니다. 학급 코드를 아는 사람이면 학급 데이터를 읽고 새 글을 쓸 수 있다는 점에서 **정식 계정 인증은 아니지만**, 화면 뒤에서 조용히 발급되는 Firebase 익명 인증 덕분에 다음은 실제로 Firestore 규칙 수준에서 강제됩니다.

- **일기 원문(사진·문장 등)은 작성한 학생 본인만 수정·삭제할 수 있어요.** 같은 반 친구나 다른 학생이 API를 직접 두드려도 막힙니다.
- 선생님은 `reviewed`(확인 여부)·코멘트 필드만 수정할 수 있고, 일기 본문은 선생님도 고칠 수 없습니다. 다만 부적절한 글 삭제(학급 관리 목적)는 선생님도 가능합니다.
- 댓글은 작성자 본인 또는 선생님만 삭제할 수 있어요.

그 외 참고할 점:
- 학급 코드와 교사 비밀번호는 안전하게 보관하고, 필요 이상으로 퍼뜨리지 마세요.
- 학생 이름은 실명 대신 별명 사용을 권장합니다. 같은 반 친구들에게 공개되는 정보이니 특히 신경 써주세요.
- 교사 인증 자체(학급 코드+비밀번호)는 여전히 계정 기반이 아니라 "비밀번호를 아는 사람" 수준입니다. 더 강한 보안(선생님 개별 로그인 등)이 필요하면 추가로 요청해주세요.
